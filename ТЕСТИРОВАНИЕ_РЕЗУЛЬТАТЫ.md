## 2. Исправленные Проблемы (Windows)

### Проблема #1: Загрузка Переменных Окружения
**Что было**: Файл `.env` не загружался до валидации  
**Что сделано**: Добавлена загрузка через `python-dotenv` в `main.py`  
**Статус**: ✅ ИСПРАВЛЕНО

### Проблема #2: Обработка Кавычек в Shell
**Что было**: Windows `cmd.exe` не понимает одинарные кавычки как bash  
**Что сделано**: 
- Замена одинарных кавычек на двойные для путей
- Специальная логика квотирования для Windows
- Изменения в `src/adapters/command.py`, `src/utils.py`, `scripts/_common.py`  
**Статус**: ✅ ИСПРАВЛЕНО

### Проблема #3: Разделители Путей
**Что было**: Обратные слеши Windows вызывали проблемы в Node.js  
**Что сделано**: Конвертация обратных слешей в прямые  
**Статус**: ✅ ИСПРАВЛЕНО

### Проблема #4: Экранирование JSON Аргументов
**Что было**: Избыточное экранирование JSON типа `[true]`  
**Что сделано**: Умное квотирование только для путей, не для JSON  
**Статус**: ✅ ИСПРАВЛЕНО

## 7. Известные Ограничения

### 7.1 Производительность Сети
**Проблема**: Bradbury Testnet медленно подтверждает транзакции  
**Влияние**: Транзакции требуют >300 секунд для подтверждения  
**Решение**: Увеличен таймаут до 300 секунд, добавлена логика ожидания появления транзакции  
**Результат**: ✅ Транзакции успешно подтверждаются при достаточном таймауте  
**Рекомендация**: Использовать таймаут 300-600 секунд для `wait_status: accepted`

### 7.2 Windows-Специфичные Особенности
**Проблема**: Квотирование в Windows shell отличается от Unix  
**Влияние**: Потребовалась специальная логика квотирования  
**Статус**: ✅ РЕШЕНО с платформо-специфичным кодом
