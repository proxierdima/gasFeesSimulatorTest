## 2. ИСПРАВЛЕННЫЕ ПРОБЛЕМЫ

### 2.1 Windows Совместимость

#### Проблема #1: Загрузка .env файла
**Симптом**: Переменные окружения не загружались до валидации  
**Решение**: Добавлена загрузка через `python-dotenv` в `main.py`  
**Файлы**: `main.py`  
**Статус**: ✅ ИСПРАВЛЕНО

#### Проблема #2: Shell Quoting
**Симптом**: Windows cmd.exe не понимает одинарные кавычки  
**Решение**: Замена одинарных кавычек на двойные для путей  
**Файлы**: `src/adapters/command.py`, `src/utils.py`, `scripts/_common.py`  
**Статус**: ✅ ИСПРАВЛЕНО

#### Проблема #3: Разделители Путей
**Симптом**: Обратные слеши Windows вызывали ошибки в Node.js  
**Решение**: Конвертация в прямые слеши  
**Файлы**: `main.py`  
**Статус**: ✅ ИСПРАВЛЕНО

#### Проблема #4: JSON Escaping
**Симптом**: Избыточное экранирование JSON аргументов  
**Решение**: Умное квотирование только для путей  
**Файлы**: `src/utils.py`, `scripts/_common.py`  
**Статус**: ✅ ИСПРАВЛЕНО

## 7. ПРОБЛЕМЫ СЕТИ BRADBURY TESTNET

### 7.1 Наблюдаемые Проблемы

#### Медленное Подтверждение
- **Симптом**: Транзакции требуют >300 секунд для подтверждения
- **Частота**: 100% случаев
- **Влияние**: Тесты превышают таймаут

#### Leader Timeout
- **Симптом**: Транзакции получают статус "LeaderTimeout"
- **Частота**: ~33% случаев
- **Влияние**: Транзакция не обрабатывается

#### Застревание в PROPOSING/COMMITTING
- **Симптом**: Транзакции застревают в промежуточных статусах
- **Частота**: ~67% случаев
- **Влияние**: Требуется очень долгое ожидание

### 7.2 Рекомендации

#### Для Тестирования
1. ✅ Использовать таймаут 300-600 секунд
2. ✅ Использовать `wait_status: accepted` вместо `finalized`
3. ✅ Запускать тесты в непиковые часы
4. ✅ Использовать mock адаптер для разработки
5. ✅ Мониторить статус сети перед запуском

#### Для Продакшн
1. Рассмотреть использование другой сети (если доступна)
2. Реализовать очередь транзакций с повторными попытками
3. Добавить алерты на длительные задержки
4. Кешировать результаты где возможно
5. Использовать асинхронную обработку
