## 🎯 БЫСТРЫЙ СТАРТ (5 МИНУТ)

### Шаг 1: Установка Зависимостей

```bash
# Python зависимости
pip install -r requirements.txt

# Node.js зависимости
npm install
```

### Шаг 2: Настройка Окружения

Создайте или отредактируйте файл `.env` в корне проекта:

```env
# Обязательные параметры
HARNESS_PRIVATE_KEY=0xВАШ_ПРИВАТНЫЙ_КЛЮЧ_ЗДЕСЬ
HARNESS_DEFAULT_NETWORK=bradbury

# Опциональные параметры
GLH_TEST_CONTRACT_FILE=contracts/fixtures/sample_contract.py
HARNESS_ADAPTER_MODE=command
HARNESS_OUTPUT_DIR=artifacts
```

⚠️ **ВАЖНО**: Замените `0xВАШ_ПРИВАТНЫЙ_КЛЮЧ_ЗДЕСЬ` на ваш реальный приватный ключ!

### Шаг 3: Запуск Тестов

```bash
# Windows
python main.py --project-root "G:\glh_v4" --password 12345678

# Linux/Mac
python main.py --project-root ~/glh_v4 --password 12345678
```

## ⚙️ НАСТРОЙКА ТАЙМАУТОВ

Если транзакции не успевают подтверждаться, увеличьте таймаут в `config/defaults.yaml`:

```yaml
timeout_seconds: 600  # Увеличить до 10 минут
poll_interval_seconds: 1.0
```

Или в конкретном сценарии (`scenarios/*/your_scenario.yaml`):

```yaml
run:
  timeout_seconds: 600
  wait_status: accepted  # Быстрее чем finalized
```

## 🐛 РЕШЕНИЕ ПРОБЛЕМ

### Проблема: "Missing required environment variables"

**Решение**: Убедитесь, что в `.env` файле установлен `HARNESS_PRIVATE_KEY`

```bash
# Проверить содержимое .env
cat .env | grep HARNESS_PRIVATE_KEY
```

### Проблема: "Transaction timed out"

**Решение**: Это нормально для Bradbury Testnet. Проверьте статус вручную:

```bash
python scripts/get_status.py --rpc https://rpc-bradbury.genlayer.com --tx <TX_HASH>
```

Если статус "Accepted" или "Finalized" - транзакция успешна!

### Проблема: "npm install fails"

**Решение**: Убедитесь, что установлен Node.js 18+

```bash
node --version  # Должно быть >= 18.0.0
npm --version   # Должно быть >= 8.0.0
```

### Проблема: "ModuleNotFoundError: No module named 'yaml'"

**Решение**: Установите Python зависимости

```bash
pip install -r requirements.txt
```
